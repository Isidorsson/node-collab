▸ 03 / architecture

how a packet travels.

node-collab is the deliberate Node twin of a Go service of mine. Same problem (multi-room realtime cursors + chat with presence), two ecosystems, two sets of trade-offs. The point is the comparison.

Two replicas sit behind a Railway router. Each replica owns a Socket.IO instance and shares fanout duties through a Redis pub/sub channel. A message published on instance A is delivered to clients connected to instance B without either knowing about the other.

Two replicas behind a router; each owns its sockets but shares fanout duties through Redis. A message published on instance A is delivered to clients connected to instance B without either knowing about the other.

▸ auth

jwt at the handshake.

Every socket presents a token in auth. The server's io.use() middleware verifies it; without a valid sub claim, the connection is rejected before any room.join.

▸ protocol

zod-validated payloads.

The five wire events — room.join, room.leave, chat, cursor, presence — share Zod schemas with the frontend's TypeScript types. Rename a field, the compiler complains on both sides.

▸ backpressure

slow clients get evicted.

Cursor packets are emitted with volatile: if the transport buffer is full, the frame is dropped, not queued. A separate watcher tracks transport-stalled timeouts and disconnects clients that can't keep up — exposed as the collab_slow_client_evictions_total Prom metric.

▸ presence

redis hashes with ttl.

In Redis mode, each room is a hash presence:<room> with a per-user TTL refresh. In memory mode (when REDIS_URL is unset), a Map plays the same role. The same PresenceStore interface fronts both.

▸ observability

prom-client over /metrics.

Active sockets, active rooms, messages published / delivered, slow-client evictions — all exposed as Prom counters and gauges, labeled by instance + channel.

▸ frontend

angular 20 standalone.

Signals for state, RxJS for event streams, a single typed CollabService mediating the socket. Lazy-loaded routes, OnPush change detection, no NgModules. Strict + strict-template TypeScript across the board.
